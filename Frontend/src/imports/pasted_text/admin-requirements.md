
RF23
El sistema debe permitir al Administrador ver el porcentaje de envíos que se encuentran a tiempo en el monitoreo en tiempo real y en las simulaciones. El cálculo es la división de envíos a tiempo totales entre la cantidad activa de envíos, multiplicado por 100%.
RF24
El sistema debe permitir al Administrador inicializar, pausar o detener la simulación semanal o la simulación hasta el colapso. Al inicializar, el Administrador puede seleccionar una fecha de inicio para la simulación mediante un selector de fecha y hora.
RF25
El sistema debe permitir al Administrador acelerar la simulación en velocidades x2, x5 y x10.
RF26
El sistema debe permitir al Administrador visualizar el estado actual de los aeropuertos de acuerdo a su porcentaje de uso actual en el monitoreo en tiempo real y en las simulaciones. Se debe diferenciar entre aeropuertos con uso normal (menor o igual a 50%), con uso moderado (mayor a 50% y menor o igual a 80%) y saturados (mayor a 80%).
RF27
El sistema debe permitir al Administrador modificar el parámetro turnaround time en el aeropuerto (para la carga/descarga).
RF28
El sistema debe permitir al Administrador subir un archivo (registrar en lote) con la información de un listado de vuelos (código del aeropuerto de origen, código del aeropuerto de destino, capacidad del avión, fecha y hora de salida, fecha y hora de llegada) para la simulación semanal y de colapso.
RF29
El sistema debe permitir al Administrador subir un archivo (registrar en lote) con la información de un listado de envíos de maletas (código del envío, cantidad de maletas, código de aeropuerto de origen, código de aeropuerto de destino y hora de registro).
RF30
El sistema debe permitir al Administrador visualizar la cantidad de envíos de maletas registrados en tiempo real.
RF31
El sistema debe permitir al Administrador visualizar la cantidad de envíos de maletas entregados en el plazo determinado en tiempo real.
RF32
El sistema debe permitir al Administrador visualizar la cantidad de envíos de maletas en tránsito en tiempo real.
RF33
El sistema debe permitir al Administrador visualizar el porcentaje de envíos que se encuentran a tiempo en tiempo real.
RF34
El sistema debe permitir al Administrador visualizar la cantidad de envíos de maletas en espera en tiempo real.
RF35
El sistema debe permitir al Administrador visualizar la cantidad de envíos de maletas fallidos en tiempo real.
RF36
El sistema debe permitir al Administrador visualizar cuando un envío de maletas ha sido registrado (cantidad de maletas, código de aeropuerto de destino, código de aeropuerto de origen, aerolínea, fecha y hora del registro) en el monitoreo en tiempo real y en las simulaciones.
RF37
El sistema debe permitir al Administrador visualizar cuando un envío de maletas despega (cantidad de maletas, código de aeropuerto de salida, código de aeropuerto de llegada, fecha y hora del registro, código de envío de maletas y código del vuelo) en el monitoreo en tiempo real y en las simulaciones.
RF38
El sistema debe permitir al Administrador visualizar cuando un envío de maletas llega al aeropuerto de escala (cantidad de maletas, código de aeropuerto de llegada, fecha y hora del registro y código de envío de maletas) en el monitoreo en tiempo real y en las simulaciones.
RF39
El sistema debe permitir al Administrador visualizar cuando un envío de maletas llega al aeropuerto de destino (cantidad de maletas, código de aeropuerto de llegada, fecha y hora del registro, código de envío de maletas y tiempo total de entrega) en el monitoreo en tiempo real y en las simulaciones.
RF40
El sistema debe permitir al Administrador visualizar cuando un envío de maletas no es entregado en su plazo establecido (código del envío de maletas, código del aeropuerto de salida, código del aeropuerto de llegada, fecha y hora del registro) en el monitoreo en tiempo real y en las simulaciones.
RF41
El sistema debe permitir al Administrador visualizar una notificación de alerta cuando la simulación llegue al colapso (un envío de maletas no llega a ser enviado en su plazo establecido) en el monitoreo en tiempo real y en las simulaciones. Esta notificación debe indicar código del envío de maletas, código del aeropuerto de salida, código del aeropuerto de llegada, fecha y hora del registro.
RF42
El sistema debe permitir al Administrador visualizar un reporte de los 8 aeropuertos con mayor ocupación actual de sus almacenes en orden descendente, mostrando además su capacidad total.
RF43
El sistema debe permitir al Administrador visualizar un reporte del uso actual (en porcentaje) respecto a la capacidad total por cada continente.
RF44
El sistema debe permitir al Administrador visualizar un reporte de las entregas acumuladas por cada fecha (de los últimos 7 días).
RF45
El sistema debe permitir al Administrador visualizar el listado de vuelos. El listado debe mostrar la siguiente información: código del aeropuerto de origen, código del aeropuerto de destino, capacidad del avión, fecha y hora de salida, tipo (intra: en un mismo continente, inter: entre diferentes continentes) y estado (activo o cancelado) para la simulación semanal y de colapso.
RF46
El sistema debe permitir al Administrador buscar un vuelo según el código del aeropuerto de origen y código del aeropuerto de destino.
RF47
El sistema debe permitir al Administrador filtrar los vuelos por el aeropuerto de origen.
RF48
El sistema debe permitir al Administrador filtrar los vuelos según su tipo (intracontinental o intercontinental).
RF49
El sistema debe permitir al Administrador filtrar los vuelos según su estado (activo o cancelado).
RF50
El sistema debe permitir al Administrador cancelar un vuelo para la simulación semanal y de colapso.
RF51
El sistema debe permitir al Administrador exportar los resultados de una simulación (KPIs de entrega, retrasos y colapso) en formato JSON o CSV.
RF52
El sistema debe permitir al Administrador visualizar un log de eventos técnicos (errores de ruteo o fallos de memoria) ocurridos durante la simulación.
RF53
El sistema debe permitir al Administrador visualizar el listado de aerolíneas activas (nombre de la aerolínea, código IATA y correo electrónico).
RF54
El sistema debe permitir al Administrador registrar una aerolínea (nombre de la aerolínea, código IATA, correo electrónico y contraseña).
RF55
El sistema debe permitir al Administrador modificar una aerolínea (nombre de la aerolínea, código IATA).
RF56
El sistema debe permitir al Administrador eliminar una aerolínea.
RF57
El sistema debe permitir al Administrador configurar alertas automáticas por correo electrónico cuando un aeropuerto supere el umbral de saturación (>80%) o cuando ocurra un colapso durante la simulación.
